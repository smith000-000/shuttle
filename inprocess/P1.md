# P1: Hardening and External Runtime Expansion

## Purpose
Track the active P1 mega-epic as a long-lived worklist spanning security/privacy hardening, shell lifecycle coverage, provider/runtime cleanup, execution-pane UX, and concrete external coding-runtime integrations.

## Current Decision Summary
- `inprocess/P1.md` is the active mega-epic tracker.
- `P#` files are mega-epic worklists, not “current top slice only” trackers.
- P0 is retired and archived as `completed/p0-agent-runtime-seam.md`.
- The lingering transcript selection rendering bug from P0 is tracked separately in GitHub and is not part of P1 acceptance.
- P1 folds in the remaining runtime lifecycle hardening and provider onboarding/ranking cleanup that were left outside the retired P0 seam work.
- Privacy posture for this epic is strict-default:
  - explicit consent for sensitive trace
  - session-scoped runtime artifacts with stale-startup pruning
  - `shuttle.log` remains operational-only, while `trace.log` owns trace-detail behavior
- External coding-runtime scope for P1 is concrete adapters behind `internal/agentruntime`, starting with `pi` and the next real runtimes, not a generalized runtime marketplace.
- Execution-pane UX scope for P1 is focused on controller-visible active execution state plus a clearer `F3` flow, not a general pane inspector.
- Shell hardening stays noninteractive-first; the interactive harness remains opt-in.

## Recent Progress
- `shuttle.log` now redacts raw commands, prompts, key input, and provider payload-like fields even when safe trace is enabled separately.
- Sensitive trace is now rejected during config parsing unless `--trace-consent` or `SHUTTLE_TRACE_CONSENT=true` is set, so env-driven startup paths cannot bypass the consent gate.
- Runtime selection now resolves `builtin`, `auto`, `pi`, and `codex_sdk` consistently, including default command filling and installed-runtime fallback.
- `internal/agentruntime` now owns named `pi` and `codex_sdk` adapter construction, while execution, approvals, patches, transcript updates, and task/session state remain controller-owned.
- TUI shutdown cleanup now follows the final recovered session name from the live model instead of always targeting the originally bootstrapped workspace session.
- Explicit proposal/approval refinement notes can now clear the active plan when the user abandons the old path or defers the remaining long-running step to another shell, preventing stale plan context and sticky plan cards from reappearing after redirection.
- Provider capability/onboarding rules now have a shared descriptor seam used by reasoning support and settings/onboarding form construction, reducing duplicated preset checks in TUI code.
- Provider-owned helpers now also drive provider ordering, display labels, model-selection validation rules, model-catalog help text, onboarding ranking, manual-provider defaults, reserved settings rows, and provider-specific model-field semantics instead of duplicating those preset checks in rendering and detection code.
- The remaining provider cleanup removed dead profile-default helper paths, so descriptor-driven profile resolution is now the only defaults path for the built-in HTTP-style providers.
- Local semantic shell state is now versioned with backward-compatible reads for older unversioned payloads, newline-terminated file validation, shell/event validation, and stale-command rejection in the state-file fallback path.
- Startup runtime cleanup now prunes stale staged command scripts, old shell-integration files, dead semantic-stream generations, and semantic state for panes no longer present in the managed tmux server, normalizes semantic artifact directories back to private permissions, and clears dead absolute managed socket files when tmux reports no live server, including missing-socket-path failures.
- Controller-owned execution overviews now drive `F3` availability, active execution card copy, and help-footer messaging so the TUI stops re-deriving owned-pane state from raw pane IDs alone.
- Provider health-test failures now surface backend-specific operator guidance instead of only echoing raw transport/auth errors.
- Plaintext provider-secret fallback now has explicit permission coverage so the secret directory stays private (`0700`) and stored secrets stay file-private (`0600`).
- Provider construction now goes through shared preset/backend helper tables instead of repeating provider registration switches in separate factory and profile paths.
- Runtime permission coverage now explicitly checks staged command dirs, shell-integration dirs, semantic artifact dirs, and operational logs in addition to provider-secret fallback files.
- The shell hardening docs and interactive harness README now explicitly call out which remote-transition, respawn, recovery, and `F2`/`F3` cases are covered by default tests versus the opt-in tty harness.
- Noninteractive shell regression coverage now explicitly pins tracked-shell migration without rewriting owned execution panes, preserves owned-pane reconcile targets after tracked-shell respawn, and keeps tracked-shell move notices visible when `RefreshActiveExecution` falls through into foreground attach.
- The prompt-return fallback path is now pinned with direct tests for semantic-exit reconciliation, remote disconnect tails that settle back to the local tracked shell without a parsed prompt, and awaiting-input tails that must not falsely reconcile.
- Controller session refresh now has direct remote-transition coverage for resolved tracked-pane aliases, remote prompt stabilization, and probe-authoritative remote directories even before a prompt is parseable again.

## Acceptance Gates
- `go test ./internal/... -count=1` must remain green throughout P1 work.
- `shuttle.log` must not retain raw commands, prompts, keys, or provider payloads.
- Sensitive trace must still require explicit consent.
- Runtime artifacts must stay private and stale artifacts must be pruned conservatively on startup/recovery.
- Semantic-state serialization must be versioned and validated without regressing current local-shell semantic behavior.
- External runtimes must remain behind `internal/agentruntime`; controller ownership boundaries must not regress.
- Provider onboarding, health checks, settings, and model listing must continue working for the current built-in backends after extensibility cleanup.
- `F2` must remain the tracked shell handoff and `F3` must remain the distinct owned execution-pane handoff when present.
- Remote transition and interactive recovery coverage must increase without requiring the interactive harness to run in the default test suite.

## Deferred / Out of Scope
- The P0 transcript selection rendering bug remains tracked in GitHub rather than through this epic.
- No generalized plugin marketplace for providers or runtimes in P1.
- No full pane inspector or multi-card/parallel execution UI in P1.
- No mandatory interactive harness in default CI; keep it opt-in unless a later epic changes that.
